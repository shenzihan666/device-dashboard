# Feature Specifications

Structured proposals and specifications for product features. Each feature follows a lifecycle from draft to shipped (or rejected).

## Lifecycle

```
Draft → Proposed → Approved → In Progress → Shipped
                 ↘ Rejected
```

| Status | Meaning |
|--------|---------|
| **Draft** | Initial writeup, not yet reviewed |
| **Proposed** | Submitted for team review |
| **Approved** | Accepted for implementation |
| **In Progress** | Active development underway |
| **Shipped** | Released and available |
| **Rejected** | Not moving forward (rationale documented) |

## Process

1. Copy the [feature spec template](../templates/feature-spec.md) into this folder.
2. Name it `NNN-short-title.md` using the next available number.
3. Fill in all sections; set status to **Draft**.
4. Present to the team; iterate until **Approved** or **Rejected**.
5. Update the index table below and in [`docs/README.md`](../README.md).
